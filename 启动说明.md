# 企业销售管理系统 - 快速启动指南

## 环境要求

### 必需环境
- **Java 17+** - 后端运行环境
- **Node.js 18+** - 前端构建环境
- **MySQL 8.0+** - 业务数据库
- **PostgreSQL 12+** - 向量数据库（可选，用于RAG）

### 可选环境
- **Maven 3.8+** - 如果不使用IDE内置Maven
- **Git** - 版本控制

## 数据库配置

### MySQL数据库
```sql
-- 1. 创建数据库
CREATE DATABASE sales_management CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 2. 使用数据库
USE sales_management;

-- 3. 执行初始化脚本
source sql/init.sql

-- 4. 插入初始数据
source sql/data.sql
```

### PostgreSQL数据库（可选）
```sql
-- 1. 创建数据库
CREATE DATABASE sales_vectors;

-- 2. 连接数据库并安装向量扩展
\c sales_vectors
CREATE EXTENSION vector;

-- 3. 执行初始化脚本（如有）
```

## 后端启动

### 方式一：使用Maven命令行

## 常见问题解决

### 1. 后端启动失败

**问题**: 数据库连接失败
```
解决方案:
1. 检查MySQL服务是否启动
2. 验证数据库连接信息
3. 确认数据库已创建
4. 检查防火墙设置
```

**问题**: 端口被占用
```bash
# 查找占用端口的进程
netstat -ano | findstr :8080

# 结束进程
taskkill /PID <进程ID> /F
```

### 2. 前端启动失败

**问题**: 依赖安装失败
```bash
# 清除缓存
npm cache clean --force

# 删除node_modules重新安装
rm -rf node_modules
npm install
```

**问题**: 端口被占用
```bash
# 修改vite.config.js中的端口
server: {
  port: 5174  # 修改为其他端口
}
```

### 3. AI功能无响应

**问题**: 大模型API调用失败
```
解决方案:
1. 检查API密钥是否有效
2. 确认API地址可访问
3. 检查网络代理设置
4. 查看后端日志错误信息
```

### 4. 文件上传失败

**问题**: OSS配置错误
```
解决方案:
1. 检查AccessKey配置
2. 确认Bucket存在且权限正确
3. 验证Endpoint配置
4. 检查文件大小限制
```

## 性能优化建议

### 后端优化
1. 启用数据库连接池
2. 配置Redis缓存
3. 启用Gzip压缩
4. 配置静态资源缓存

### 前端优化
1. 启用代码分割
2. 使用CDN加速
3. 启用图片懒加载
4. 配置浏览器缓存

## 安全建议

1. 修改默认密码
2. 启用HTTPS
3. 配置CORS白名单
4. 定期更新依赖
5. 备份数据库
