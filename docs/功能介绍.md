# 功能介绍

## 概述
企业销售管理系统是一个集成了销售业务管理、数据分析和智能问答的综合性平台。系统采用前后端分离架构，后端基于Spring Boot + MyBatis Plus + LangChain4j，前端基于Vue 3 + Element Plus，提供完整的销售业务管理解决方案。

## 系统架构

### 整体架构
```
┌─────────────────────────────────────────────────────────────┐
│                      前端 (Vue 3)                          │
├─────────────────────────────────────────────────────────────┤
│                    RESTful API                              │
├─────────────────────────────────────────────────────────────┤
│                 Spring Boot 后端                            │
├───────────────┬───────────────┬───────────────┬─────────────┤
│   安全模块    │   业务模块    │   AI模块      │   存储模块  │
├───────────────┼───────────────┼───────────────┼─────────────┤
│ Spring Security│ MyBatis Plus │ LangChain4j  │ 阿里云OSS   │
├───────────────┼───────────────┼───────────────┼─────────────┤
│    JWT        │    MySQL      │   文件系统    │   缓存      │
└───────────────┴───────────────┴───────────────┴─────────────┘
```

### 技术栈详情
- **前端**: Vue 3 + Element Plus + Axios + ECharts
- **后端**: Spring Boot 3.2 + MyBatis Plus 3.5 + LangChain4j 0.25
- **数据库**: MySQL 8.0
- **存储**: 阿里云OSS
- **安全**: Spring Security + JWT
- **文档**: Apache POI (PDF生成)

## 功能模块详解

### 1. 总览模块

#### 功能概述
总览模块是系统的数据可视化中心，提供销售业务的整体视图，帮助管理者快速了解业务状况。

#### 核心功能
1. **数据统计面板**
   - 总销售额：系统历史累计销售额
   - 月度销售额：当前月度销售额
   - 日均销售额：平均每日销售额
   - 订单总数：系统历史订单总数
   - 待处理订单：当前待处理订单数量
   - 进行中订单：当前进行中订单数量
   - 已完成订单：当前已完成订单数量

2. **销售排行榜**
   - 销冠展示：显示销售额最高的销售人员
   - 销售排名：按销售额排序的销售人员列表
   - 个人业绩：每个销售人员的详细业绩数据
   - 照片展示：销售人员照片和基本信息

3. **热门商品TOP10**
   - 销量排名：按销量排序的商品列表
   - 销售额排名：按销售额排序的商品列表
   - 商品详情：商品图片、名称、价格等信息
   - 趋势分析：商品销售趋势图表

4. **订单状态分布**
   - 饼图展示：各状态订单占比
   - 状态统计：各状态订单数量
   - 趋势分析：订单状态变化趋势

5. **销售趋势图表**
   - 时间维度：按日/周/月展示销售趋势
   - 对比分析：同期对比、环比分析
   - 图表类型：折线图、柱状图、面积图

6. **PDF报表导出**
   - 总览报表：包含所有统计数据的PDF报表
   - 自定义报表：可选择导出内容
   - 定时报表：支持定时生成报表

#### 使用场景
- **每日晨会**：快速了解昨日销售情况
- **周度总结**：分析一周销售趋势
- **月度报告**：生成月度销售报告
- **年度分析**：年度销售数据分析

#### 数据更新频率
- 实时数据：订单状态、销售额等
- 定时更新：排行榜、热门商品（每小时更新）
- 手动刷新：支持手动刷新数据

### 2. 商品管理模块

#### 功能概述
商品管理模块提供完整的商品生命周期管理，包括商品信息维护、库存管理、图片管理等。

#### 核心功能
1. **商品信息管理**
   - 商品列表：分页展示所有商品
   - 商品搜索：支持按名称、分类搜索
   - 商品筛选：按价格区间、库存状态筛选
   - 批量操作：支持批量上架、下架、删除

2. **商品详情编辑**
   - 基本信息：商品名称、描述、价格
   - 库存管理：库存数量、预警阈值
   - 图片管理：商品图片上传、预览、删除
   - 分类管理：商品分类标签

3. **图片上传功能**
   - OSS集成：阿里云OSS对象存储
   - 图片压缩：自动压缩优化图片
   - 多图支持：支持上传多张图片
   - 图片预览：实时预览上传效果
   - 格式支持：JPG、PNG、GIF、WEBP

4. **库存管理**
   - 库存查看：实时查看库存数量
   - 库存预警：低库存预警提醒
   - 库存调整：手动调整库存数量
   - 库存日志：库存变动记录

5. **商品分类**
   - 分类管理：创建、编辑、删除分类
   - 分类筛选：按分类筛选商品
   - 分类统计：各分类商品数量统计

#### 使用流程
1. **添加商品**
   - 填写商品基本信息
   - 上传商品图片
   - 设置库存数量
   - 选择商品分类
   - 保存商品信息

2. **编辑商品**
   - 修改商品信息
   - 更新商品图片
   - 调整库存数量
   - 保存修改

3. **删除商品**
   - 确认删除操作
   - 逻辑删除（软删除）
   - 关联数据处理

#### 权限控制
- **管理员**：完整权限
- **销售**：只读权限（无增删改权限）

### 3. 订单管理模块

#### 功能概述
订单管理模块处理销售订单的整个生命周期，从创建到完成，包括状态管理、库存扣减、PDF生成等。

#### 核心功能
1. **订单创建**
   - 客户信息：客户名称、联系方式
   - 销售人员：选择销售人员
   - 商品选择：从商品库选择商品
   - 数量设置：设置商品数量
   - 价格计算：自动计算总价
   - 订单预览：创建前预览订单

2. **订单状态管理**
   - 状态流转：待处理→进行中→已完成/已取消
   - 状态变更：手动变更订单状态
   - 状态日志：记录状态变更历史
   - 批量操作：批量更新订单状态

3. **订单详情**
   - 基本信息：订单号、客户、销售、时间
   - 商品列表：订单包含的商品详情
   - 金额信息：商品单价、数量、总价
   - 状态信息：当前状态、变更历史

4. **库存扣减**
   - 自动扣减：订单完成时自动扣减库存
   - 库存检查：创建订单时检查库存
   - 库存回滚：订单取消时回滚库存
   - 库存预警：低库存预警提醒

5. **PDF生成**
   - 订单PDF：生成订单详情PDF
   - 格式规范：标准商务PDF格式
   - 自定义模板：支持自定义PDF模板
   - 批量生成：批量生成订单PDF

6. **订单搜索**
   - 关键词搜索：按订单号、客户名搜索
   - 状态筛选：按订单状态筛选
   - 时间筛选：按时间范围筛选
   - 销售筛选：按销售人员筛选

#### 订单状态说明
1. **待处理(PENDING)**
   - 新创建的订单
   - 等待销售人员处理
   - 可以修改订单信息

2. **进行中(IN_PROGRESS)**
   - 销售人员已接单
   - 正在处理中
   - 可以更新进度

3. **已完成(COMPLETED)**
   - 订单处理完成
   - 自动扣减库存
   - 可以生成PDF

4. **已取消(CANCELLED)**
   - 订单被取消
   - 回滚库存（如果已扣减）
   - 记录取消原因

#### 使用流程
1. **创建订单**
   - 选择客户
   - 选择销售人员
   - 添加商品
   - 设置数量
   - 确认创建

2. **处理订单**
   - 查看订单详情
   - 更新订单状态
   - 跟踪处理进度

3. **完成订单**
   - 确认订单完成
   - 系统自动扣减库存
   - 生成订单PDF

4. **取消订单**
   - 确认取消原因
   - 系统回滚库存
   - 记录取消信息

### 5. 智能问答模块

#### 功能概述
智能问答模块是基于AI的智能数据分析系统，支持自然语言提问，自动生成SQL查询，对查询结果进行智能分析。系统使用大语言模型（LLM）进行意图识别、SQL生成和数据分析。

#### 核心功能
1. **意图分析**
   - 问题分类：自动识别问题类型
   - 实体识别：提取问题中的关键实体
   - 意图理解：理解用户真实意图
   - 上下文理解：理解对话上下文

2. **SQL生成功能**
   - 自然语言转SQL：将问题转换为SQL语句
   - 表结构理解：理解数据库表结构
   - 查询优化：生成优化的SQL查询
   - 安全检查：SQL注入防护

3. **数据库查询**
   - 自动执行：自动执行生成的SQL
   - 结果处理：处理查询结果
   - 错误处理：处理查询错误
   - 性能优化：查询性能优化

4. **AI分析整合**
   - 数据分析：对查询结果进行分析
   - 趋势识别：识别数据趋势
   - 异常检测：检测数据异常
   - 建议生成：生成业务建议

5. **流式输出**
   - SSE支持：使用Server-Sent Events进行流式输出
   - 实时显示：实时显示AI回答
   - 逐字输出：逐字输出回答内容
   - 打字机效果：打字机效果显示

6. **对话管理**
   - 对话历史：保存对话历史
   - 上下文维护：维护对话上下文
   - 多轮对话：支持多轮对话
   - 对话清除：清除对话历史

#### 支持的意图类型
1. **sales_query** - 销售数据查询
   - 示例："本月销售额最高的销售是谁？"
   - 示例："本月销售情况如何？"

2. **product_query** - 商品数据查询
   - 示例："哪些商品销量最好？"
   - 示例："库存不足的商品有哪些？"

3. **order_query** - 订单数据查询
   - 示例："本月有多少订单？"
   - 示例："已完成的订单有哪些？"

4. **employee_query** - 员工数据查询
   - 示例："张三的业绩怎么样？"
   - 示例："销售团队有多少人？"

5. **statistics** - 统计分析
   - 示例："本月销售总额是多少？"
   - 示例："订单完成率是多少？"

6. **comparison** - 对比分析
   - 示例："本月和上月销售对比如何？"
   - 示例："哪个销售业绩最好？"

7. **trend** - 趋势分析
   - 示例："销售趋势如何？"
   - 示例："订单量有什么变化？"

8. **general** - 通用问答
   - 示例："你好"
   - 示例："系统有什么功能？"

#### 使用示例
1. **销售数据分析**
   - 问题："本月销售额最高的销售是谁？"
   - 分析：意图识别为销售数据查询
   - SQL生成：`SELECT u.name, SUM(o.total_amount) FROM orders o JOIN sys_user u ON o.salesperson_id = u.id WHERE MONTH(o.create_time) = MONTH(CURRENT_DATE()) GROUP BY u.id ORDER BY SUM(o.total_amount) DESC LIMIT 1`
   - 答案："根据数据分析，本月销售额最高的销售是张三，总销售额为50000元。"

2. **商品销售分析**
   - 问题："哪些商品销量最好？"
   - 分析：意图识别为商品销售分析
   - SQL生成：`SELECT p.name, SUM(oi.quantity) FROM order_item oi JOIN product p ON oi.product_id = p.id GROUP BY p.id ORDER BY SUM(oi.quantity) DESC LIMIT 10`
   - 答案："销量最好的商品是iPhone 15，共销售100件。"

3. **库存分析**
   - 问题："哪些商品库存不足？"
   - 分析：意图识别为库存查询
   - SQL生成：`SELECT name, stock_quantity FROM product WHERE stock_quantity < 10`
   - 答案："库存不足的商品有：iPhone 15（库存5件）、MacBook Pro（库存3件）。"
