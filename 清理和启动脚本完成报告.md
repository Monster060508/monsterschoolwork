## 创建的启动脚本

### 1. 完整启动脚本 (`start.bat`)

**功能特性：**
- 交互式菜单界面
- 支持多种启动方式
- 自动检查依赖环境
- 端口占用检测和处理
- 数据库初始化功能
- 测试运行功能

**菜单选项：**
1. 启动完整系统（后端 + 前端）
2. 仅启动后端
3. 仅启动前端
4. 初始化数据库
5. 运行测试
6. 退出

### 2. 快速启动脚本 (`quick-start.bat`)

**功能特性：**
- 一键启动整个系统
- 自动等待后端启动（30秒）
- 自动打开浏览器
- 显示访问信息和测试账号

**使用场景：**
- 快速启动开发环境
- 演示系统功能
- 日常开发使用

### 3. 停止服务脚本 (`stop.bat`)

**功能特性：**
- 停止所有Java进程（后端）
- 停止所有Node进程（前端）
- 安全终止服务

**使用场景：**
- 停止所有运行中的服务
- 端口冲突时清理进程
- 开发结束后清理环境

## 使用指南

### 首次使用
1. 复制 `.example` 文件创建实际配置：
   ```bash
   cp 配置信息.md.example 配置信息.md
   cp docs/配置文档.md.example docs/配置文档.md
   ```

2. 编辑配置文件，填入真实的配置信息

3. 运行 `quick-start.bat` 启动系统

### 日常开发
- 使用 `start.bat` 进行开发
- 使用 `stop.bat` 停止服务
- 定期运行测试确保功能正常

### 生产部署
1. 使用 `start.bat` 的选项4初始化数据库
2. 使用 `start.bat` 的选项5运行测试
3. 参考 `启动说明.md` 进行生产环境部署

## 注意事项

1. **敏感信息保护**
   - 不要将包含真实密码的文件提交到Git
   - 使用环境变量管理生产环境配置
   - 定期更换API密钥和密码

2. **端口管理**
   - 默认后端端口：8080
   - 默认前端端口：5173
   - 如有冲突，脚本会提示处理

3. **依赖要求**
   - Java 17+
   - Node.js 18+
   - Maven 3.8+
   - MySQL 8.0+

4. **日志查看**
   - 后端日志：控制台输出
   - 前端日志：浏览器开发者工具
   - 错误排查：查看对应窗口的错误信息
