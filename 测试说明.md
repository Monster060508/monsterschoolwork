# 企业销售管理系统 - 测试说明

## 系统启动

### 1. 数据库初始化
```sql
-- 执行初始化脚本
source sql/init.sql

-- 插入初始数据
source sql/data.sql
```

### 2. 启动后端服务
```bash
cd backend
mvn spring-boot:run
```

### 3. 启动前端服务
```bash
cd frontend
npm install
npm run dev
```

### 4. 访问系统
- 前端地址：http://localhost:5173
- 后端API：http://localhost:8080/api


#### 流式聊天（SSE）
```bash

  -H "Content-Type: application/json" \

```

#### 获取对话历史
```bash
curl http://localhost:8080/api/ai/conversations

# 获取特定对话的历史
curl http://localhost:8080/api/ai/conversation/test-001

# 清除对话历史
curl -X DELETE http://localhost:8080/api/ai/conversation/test-001
```

## SSE流式输出说明

### 前端调用示例
```javascript
import { aiApi } from '@/api'

// 流式聊天
aiApi.chatStream(
  { question: '本月销售情况如何？', conversationId: 'conv-001' },
  // onToken - 每收到一个token触发
  (token) => {
    console.log('收到token:', token)
    // 更新UI显示
  },
  // onComplete - 完成时触发
  (response) => {
    console.log('完成:', response)
    // 处理完成逻辑
  },
  // onError - 错误时触发
  (error) => {
    console.error('错误:', error)
    // 处理错误
  }
)
```

### SSE事件格式
```
event:conversationId
data:conv-001

event:token
data:本月

event:token
data:销售

event:token
data:情况

event:complete
data:{"fullResponse":"本月销售情况...","intent":"sales_query","path":"SQL"}
```

## 常见问题

### 1. 数据库连接失败
检查 `application.yml` 中的数据库配置：
- MySQL: `jdbc:mysql://47.103.77.247:3306/sales_management`
- PostgreSQL: `jdbc:postgresql://47.103.77.247:5432/sales_vectors`

### 2. AI接口无响应
检查大模型API配置：
- API地址：`https://token-plan-cn.xiaomimimo.com/v1`
- API密钥：配置文件中的 `api-key`
- 模型名称：`mimo-v2.5-pro`

### 3. 文件上传失败
检查OSS配置：
- Endpoint: `https://oss-cn-hangzhou.aliyuncs.com`
- Bucket: `monster060508`
- AccessKey配置正确

### 4. 前端无法访问后端API
检查CORS配置和代理设置：
- 后端允许的源：`http://localhost:5173`, `http://localhost:3000`
- Vite代理配置正确
