# 启动脚本说明

## 脚本文件列表

| 脚本文件 | 功能说明 |
|---------|---------|
| `start.bat` | 完整启动脚本，提供多种启动选项 |
| `quick-start.bat` | 快速启动脚本，一键启动整个系统 |
| `stop.bat` | 停止所有服务的脚本 |

## 使用方法

### 1. 完整启动脚本 (start.bat)

双击运行 `start.bat`，将显示菜单选项：

```
========================================
   企业销售管理系统 - 启动脚本
========================================

请选择启动方式:

1. 启动完整系统（后端 + 前端）
2. 仅启动后端
3. 仅启动前端
4. 初始化数据库
5. 运行测试
6. 退出
```

**功能说明：**
- **选项1**: 同时启动后端和前端服务
- **选项2**: 仅启动后端Spring Boot服务
- **选项3**: 仅启动前端Vue开发服务器
- **选项4**: 运行数据库初始化脚本
- **选项5**: 运行各种测试（单元测试、API测试等）
- **选项6**: 退出脚本

### 2. 快速启动脚本 (quick-start.bat)

双击运行 `quick-start.bat`，将自动：

1. 启动后端服务（在新窗口）
2. 等待30秒让后端完全启动
3. 启动前端服务（在新窗口）
4. 显示访问信息
5. 按任意键打开浏览器访问系统

**访问地址：**
- 前端: http://localhost:5173
- 后端: http://localhost:8080

**测试账号：**
- 管理员: `admin` / `admin123`
- 销售: `sales001` / `sales123`

### 3. 停止服务脚本 (stop.bat)

双击运行 `stop.bat`，将：

1. 停止所有Java进程（后端服务）
2. 停止所有Node进程（前端服务）

**注意**: 此脚本会终止所有Java和Node进程，请确保没有其他重要程序在运行。

## 前置要求

### 必需软件
- **Java 17+**: 后端运行环境
- **Node.js 18+**: 前端构建环境
- **Maven 3.8+**: 后端依赖管理
- **MySQL 8.0+**: 业务数据库

### 可选软件
- **Python 3.8+**: 用于运行测试脚本
- **Git**: 版本控制

## 首次运行步骤

1. **安装依赖软件**
   - 安装Java JDK 17或更高版本
   - 安装Node.js 18或更高版本
   - 安装Maven 3.8或更高版本
   - 安装MySQL 8.0或更高版本

2. **配置数据库**
   - 启动MySQL服务
   - 创建数据库: `CREATE DATABASE sales_management;`
   - 执行初始化脚本: `sql/init.sql`
   - 插入初始数据: `sql/data.sql`

3. **配置后端**
   - 编辑 `backend/src/main/resources/application.yml`
   - 配置数据库连接信息
   - 配置其他必要参数（OSS、大模型API等）

4. **启动系统**
   - 双击运行 `quick-start.bat`
   - 或运行 `start.bat` 选择启动方式

## 常见问题

### Q1: 启动时提示"找不到Maven"
**解决方案：**
1. 下载并安装Maven: https://maven.apache.org/download.cgi
2. 将Maven的`bin`目录添加到系统PATH环境变量
3. 或者使用IDE内置的Maven

### Q2: 启动时提示"找不到Node.js"
**解决方案：**
1. 下载并安装Node.js: https://nodejs.org/
2. 安装时勾选"Add to PATH"选项
3. 重启命令行窗口

### Q3: 后端启动失败，提示数据库连接错误
**解决方案：**
1. 检查MySQL服务是否启动
2. 验证 `application.yml` 中的数据库配置
3. 确保数据库 `sales_management` 已创建
4. 检查数据库用户名和密码是否正确

### Q4: 前端启动失败，提示端口被占用
**解决方案：**
1. 运行 `stop.bat` 停止所有服务
2. 或手动终止占用端口的进程:
   ```bash
   netstat -ano | findstr :5173
   taskkill /PID <进程ID> /F
   ```

### Q5: 启动后无法访问系统
**解决方案：**
1. 等待足够时间（后端启动约需30-60秒）
2. 检查防火墙设置
3. 查看控制台是否有错误信息
4. 尝试单独启动后端和前端进行排查

## 开发模式启动

如果需要进行开发，建议使用以下方式：

### 后端开发
```bash
cd backend
mvn spring-boot:run
```

### 前端开发
```bash
cd frontend
npm install  # 首次运行
npm run dev
```

### 同时开发
1. 打开两个命令行窗口
2. 窗口1: 运行后端
3. 窗口2: 运行前端
4. 使用IDE进行代码编辑和调试

## 生产环境部署

### 后端部署
```bash
cd backend
mvn clean package -Pprod
java -jar target/sales-management-1.0.0.jar --spring.profiles.active=prod
```

### 前端部署
```bash
cd frontend
npm run build
# 将dist目录部署到Web服务器
```
